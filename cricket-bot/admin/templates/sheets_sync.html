⚠️ Not Configured

Google Sheets integration requires three environment variables:

VariableDescription
GOOGLE_SHEETS_CREDENTIALS_FILE Path to the service-account JSON key file downloaded from the Google Cloud Console.
GOOGLE_SHEETS_SPREADSHEET_ID The spreadsheet ID or full URL. Share the sheet with the service-account email (Editor access).
GOOGLE_SHEETS_WORKSHEET_NAME Tab / worksheet name inside the spreadsheet (default: Players).
Quick Setup Steps
  1. Go to Google Cloud Console → create a project (or use an existing one).
  2. Enable the Google Sheets API and Google Drive API.
  3. Create a Service Account under IAM & Admin → Service Accounts.
  4. Generate a JSON key for the service account and download it.
  5. Create a Google Sheet (or use an existing one) and share it with the service-account email address (e.g. bot@my-project.iam.gserviceaccount.com) as Editor.
  6. Set the three env vars above and restart the bot.
{% else %}
⬆️ Export to Google Sheets

Push all players from the database into the configured Google Sheet. The sheet will be cleared first and replaced with current data.

  • Overwrites the entire sheet
  • Header row is added automatically
  • Players sorted by rating (highest first)
⬇️ Import from Google Sheets

Read rows from the Google Sheet and sync them into the database.

  • Rows with an existing id → updated
  • Rows with empty id → created (if name is unique)
  • Duplicate names and invalid rows are skipped
💡 Recommended Workflow
  1. Export players to the Google Sheet.
  2. Edit player data directly in Google Sheets — update ratings, add new rows, fix stats, etc.
  3. Import the sheet back to apply all your changes at once.

You can also share the sheet with your team so multiple people can edit player data before importing.
